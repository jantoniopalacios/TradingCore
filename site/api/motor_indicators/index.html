    Referencia API: Indicadores Técnicos

    Este módulo contiene la implementación y la lógica de filtrado específica para cada indicador técnico.

    Filtro EMA (Exponential Moving Average)

    Módulo para la lógica de la Media Móvil Exponencial (EMA).

    Contiene funciones delegadas para la actualización del estado dinámico (ascendente/descendente/mínimo/máximo) de la EMA Lenta, y la generación de señales de compra/venta basadas en cruces y tendencia.

    update_ema_state(strategy_self, verificar_estado_indicador_func)

    Actualiza el estado dinámico (STATE) de la EMA Lenta (ma_slow) basado en sus últimos valores.

    Este estado se almacena como variables booleanas de instancia (ej. ema_slow_minimo_STATE) para ser utilizado posteriormente por las funciones de señal y filtrado.

    Parameters:

    Name Type Description Default
    strategy_self System

    Instancia de la clase System (Strategy), que accede a self.ema_slow_series.

    		 required
    verificar_estado_indicador_func CheckStateFunc

    Función auxiliar que calcula el estado dinámico (minimo, maximo, ascendente, descendente) de una serie de pandas.

    		 required

    Returns:

    Type Description
    None

    Actualiza los atributos de estado de la instancia strategy_self directamente.

    check_ema_buy_signal(strategy_self, condicion_base_tecnica)

    Revisa las señales de compra (lógica OR) generadas por la EMA (Cruce, Mínimo o Ascendente).

    Cualquiera de estas señales puede establecer la condicion_base_tecnica como True.

    Parameters:

    Name Type Description Default
    strategy_self System

    Instancia de la estrategia, que contiene las series EMA y las configuraciones del usuario.

    		 required
    condicion_base_tecnica bool

    Condición técnica global actual (el resultado de las señales OR anteriores).

    		 required

    Returns:

    Type Description
    tuple
    • bool: Nueva condicion_base_tecnica (True si se activó una señal OR).
    • Optional[str]: Razón del log de la señal activada, o None si no hay señal de EMA.

    apply_ema_global_filter(strategy_self, condicion_base_tecnica)

    Aplica el filtro global excluyente (Condición AND) de la EMA Lenta.

    Este filtro puede anular cualquier señal de compra generada por la lógica OR si las condiciones de tendencia de la EMA Lenta son desfavorables o si no se cumplen los requerimientos mínimos.

    Parameters:

    Name Type Description Default
    strategy_self System

    Instancia de la estrategia con los parámetros de filtrado activados.

    		 required
    condicion_base_tecnica bool

    Condición técnica global actual.

    		 required

    Returns:

    Type Description
    bool

    La condicion_base_tecnica modificada (False si falla el filtro, True en caso contrario).

    check_ema_sell_signal(strategy_self)

    Revisa la señal de venta de cierre técnico basada en el estado de la EMA Lenta.

    La señal de venta se activa si la EMA Lenta pasa a un estado de tendencia descendente, sirviendo como filtro de salida de tendencia.

    Parameters:

    Name Type Description Default
    strategy_self System

    Instancia de la estrategia.

    		 required

    Returns:

    Type Description
    tuple
    • bool: True si la señal de venta está activa.
    • Optional[str]: Descripción del cierre para el log, o None.

    Filtro RSI (Relative Strength Index)

    Módulo para la lógica del Índice de Fuerza Relativa (RSI).

    Contiene funciones delegadas para la actualización del estado dinámico (ascendente/descendente/mínimo/máximo) del RSI y la generación de señales de compra/venta basadas en giros y niveles de fuerza.

    update_rsi_state(strategy_self, verificar_estado_indicador_func)

    Actualiza el estado dinámico (STATE) del indicador RSI.

    Calcula y almacena el estado dinámico (mínimo, máximo, ascendente, descendente) en las variables de instancia strategy_self si el RSI está activo.

    Parameters:

    Name Type Description Default
    strategy_self System

    Instancia de la clase System (Strategy), que accede a self.rsi_ind.

    		 required
    verificar_estado_indicador_func CheckStateFunc

    Función auxiliar que calcula el estado dinámico de una serie de pandas.

    		 required

    Returns:

    Type Description
    None

    Actualiza los atributos de estado de la instancia strategy_self directamente.

    check_rsi_buy_signal(strategy_self, condicion_base_tecnica)

    Revisa las señales de compra (lógica OR) generadas por el RSI (Giro de Mínimo o Fuerza Pura).

    La señal de compra se activa por un giro desde un extremo (sobreventa) o por una fuerza continua por encima de un umbral predefinido.

    Parameters:

    Name Type Description Default
    strategy_self System

    Instancia de la estrategia, que contiene las series RSI y las configuraciones del usuario.

    		 required
    condicion_base_tecnica bool

    Condición técnica global actual (el resultado de las señales OR anteriores).

    		 required

    Returns:

    Type Description
    tuple
    • bool: Nueva condicion_base_tecnica (True si se activó una señal OR).
    • Optional[str]: Razón del log de la señal activada, o None si no hay señal de RSI.

    check_rsi_sell_signal(strategy_self)

    Revisa la señal de venta de cierre técnico basada en el estado de sobrecompra o tendencia descendente del RSI.

    La señal de venta se activa si la estrategia está configurada para reaccionar a niveles máximos o a un cambio de tendencia descendente en el RSI.

    Parameters:

    Name Type Description Default
    strategy_self System

    Instancia de la estrategia.

    		 required

    Returns:

    Type Description
    tuple
    • bool: True si la señal de venta está activa.
    • Optional[str]: Descripción del cierre para el log, o None.
